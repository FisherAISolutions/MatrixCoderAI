# Matrix Coder AI Product North Star

## Mission

Help anyone turn an idea into trustworthy, production-ready software through a guided conversation with an AI software engineering team.

## User-Facing Journey

Tell Matrix your idea
-> natural back-and-forth discovery
-> Matrix recommends helpful improvements
-> user accepts or rejects suggestions
-> Matrix reaches a complete understanding
-> user approves the plan
-> Matrix builds in bounded tasks
-> preview and conversational changes
-> evidence-based completion
-> launch

## Internal Architecture

Natural Conversation
-> Matrix Intelligence Core
-> Architect Draft
-> Blueprint
-> Build Contract
-> Capabilities
-> Task Graph
-> Repository-Aware Execution
-> Task-Level Validation and Repair
-> Evidence-Based Final Review
-> Preview, Changes, and Deployment

## Permanent Product Rules

- Conversation is the primary interface.
- The user's approved vision is the source of truth.
- Matrix never attempts a large build from a vague prompt.
- Matrix builds one bounded engineering task at a time.
- Repository state is checked before every coding task.
- Completed work is preserved after failure.
- A build is not complete just because it compiles.
- Every required feature needs real evidence.
- Technical complexity is hidden by default but available to advanced users.
- Matrix recommends based on the user's goals, budget, and experience.
- Matrix remains general-purpose and is not limited to known templates.
- Every prompt and commit must move directly toward the final experience.

## Scope Guard

Unrelated feature expansion, new app-type agents, visual novelty, and parallel memory or planning systems are postponed unless they directly support this mission.

## Implementation Test

"Does this change make Matrix better at guiding, remembering, building carefully, proving quality, and helping the user launch?"
